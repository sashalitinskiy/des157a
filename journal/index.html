A user is much more likely to be committed to some action if they are drawn in with the easier tasks first. A website that I consider to have mastered these best practices is the Airbnb website. Booking a rental home is a difficult, big task, that comes with 10s of smaller subtasks. The Airbnb website knows this and makes it so that everything is very purposeful. It is made with the user not only in mind, but in the center of design and usability. Doing something very stressful is made smooth and even exciting by the way that the website is designed. Every inch on the screen serves a purpose.

Learning Journal 2

This article talked about the best practices for overlays/modals/dialogs. First off, I'll give the definition of a modal window. A modal is something that is on top of the application's main window, disabling the parent content. It makes the UI simpler while saving space on screen. It is commonly used when designers need to capture a user's full attention by interrupting their current task. Other use cases include collecting input and displaying additional information. Their overuse has made them seem frustrating as pop-ups. This leads to users dismissing them, I often do the same! A best practice includes showing a clear exit option such as a close button, escape key, etc. Another is using actionable button labels. A third is using good sizing to avoid overwhelming the entire screen. TWhat I found the article was saying is that, most importantly, accessibility is the main focus. Modals should enhance the user experience, not take it away.
